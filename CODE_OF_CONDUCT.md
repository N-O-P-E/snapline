# Code of Conduct

## Our standard

We expect everyone interacting in Snapline's repos, issue tracker, pull requests, and any other forum to be respectful, constructive, and on-topic.

That means:

- Treat people the way you'd want to be treated. Disagreement is fine; personal attacks are not.
- Keep technical discussions focused on the work, not the person doing it.
- Assume good intent. If a comment reads as harsh, ask for clarification before escalating.

We will not tolerate harassment, discrimination, or persistently abusive behaviour. Maintainers reserve the right to remove, edit, or reject comments, commits, code, issues, and contributions that don't align with this policy, and to ban (temporarily or permanently) anyone who violates it.

## Reporting

If someone's behaviour is making the project unwelcoming, email **conduct@studionope.nl**. Reports stay confidential.

This document is intentionally short — it's a living document and we'll expand it if specific situations demand more nuance.
